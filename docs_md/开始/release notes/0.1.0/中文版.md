# The One 0.1.0 Release Notes

The One 0.1.0 是我们的第一个桌面版本，也是这款产品理念的首次完整落地：让用户不再为了探索 Multi-Agents 架构反复写胶水代码，而是直接用一款桌面应用搭建、运行、观察和迭代自己的多智能体系统。

这不是又一个单 Agent 聊天窗口。The One 从第一天就是为 Multi-Agents 设计的。

## 核心理念

多智能体是 AI Agent 的未来，但今天的大多数探索还停留在“手写几个 Agent、硬编码几条调用链”的阶段。这个过程慢、脆弱、不可复用，也很难看清 Agent 之间到底发生了什么。

The One 的目标很直接：用一款软件来搭建任意人们能想象的 Multi-Agents 架构。

## 新增功能

### Multi-Agents 仓储系统

0.1.0 提供了一套面向多智能体架构的资源仓储界面：

- Agent Preset：定义不同 Agent 的身份、模型、提示词、Profile、Skill 与工具能力。
- Profile / Skill：将长提示词、可复用能力说明和上下文模块独立维护，按需挂载到不同 Agent。
- Tool / MCP：为 Agent 配置内置工具或外部 MCP 工具，让能力可以被组合，而不是写死在代码里。
- Graph Blueprint：通过节点和连线搭建 Multi-Agents Graph，定义 Agent 之间的协作关系与通讯路径。

你可以先打磨 Agent，再组装 Graph。这个顺序很重要，因为好的多智能体系统不是“堆几个模型”，而是明确角色、能力边界和协作协议。

### 可视化 Graph 蓝图编辑器

The One 0.1.0 支持用画布方式创建和编辑 Multi-Agents Graph：

- 在画布中添加 Agent 节点。
- 为每个节点绑定不同的 Agent Preset。
- 通过边连接 Agent，表达协作、交接、反馈或审核关系。
- 为特定节点追加专属上下文，让同一个 Agent Preset 在不同 Graph 中承担不同角色。
- 创建、重命名、删除和保存 Workflow Blueprint。

这让多智能体架构从“藏在代码里的流程”变成“可以被看见、讨论和修改的系统设计”。

### 本地项目工作区运行

0.1.0 支持在指定项目路径下启动 Workflow，让你创建的 Multi-Agents Graph 真正进入工作状态。

你可以选择一个项目目录，创建一次 Workflow 运行，然后由你设计的 Agent 网络围绕这个工作区展开协作。这个体验更接近本地桌面 Agent 应用，但核心区别是：运行的不是固定 Agent，而是你自己搭出来的 Multi-Agents Graph。

### 透明的运行观察界面

The One 非常明确地反对黑箱式 Agent 运行。

在 0.1.0 中，你可以从“上帝视角”观察一次 Graph 运行：

- 查看当前 Workflow 的执行链路。
- 查看运行中的 Agent 列表与状态。
- 进入单个 Agent 的消息历史。
- 查看 Agent 的上下文组成。
- 查看 Agent 可用工具与工具调用记录。
- 查看共享数据与运行快照。
- 查看子 Workflow、Task Board、任务层级和任务结果。

这套透明性不是装饰功能，而是 Multi-Agents 探索的基础设施。看不清 Agent 怎么协作，就不可能系统性改进架构。

### 面向复杂任务的 Task Board 协作

0.1.0 引入了面向复杂任务拆解的 Task Board 能力。

Agent 可以将任务拆成分层任务板：同一层中的任务可以并行执行，后续层级可以等待前置结果完成后继续推进。任务执行过程中可以创建 Worker / Reviewer 式的协作结构，让复杂任务不再只是一个 Agent 从头干到尾。

这让 The One 不只是“连几个 Agent”，而是开始支持真正有组织的多角色、多阶段协作。

### 动态 Graph 能力

The One 0.1.0 已经具备运行时动态协作的基础能力：

- Agent 可以唤醒其他节点。
- Agent 可以让自己或其他节点进入等待状态。
- Agent 可以完成 Workflow。
- Agent 可以读取 Workflow 节点状态。
- Agent 可以在运行中扩展协作关系。

这意味着 Graph 不必永远是静态流程图。随着任务推进，Agent 可以根据实际情况调整协作结构。这是 The One 区别于传统工作流工具的关键方向。

### 人类参与机制

0.1.0 支持 Agent 在需要时向人类请求输入，包括：

- 提问
- 确认
- 选择
- 向指定 Agent 发送补充上下文
- 在等待人类回复时保持运行状态可观察

这避免了 Agent 卡死在不确定决策上，也让人类可以在关键节点介入，而不是只能从头到尾旁观。

### 模型、语言与桌面体验

0.1.0 还提供了完整桌面应用所需的基础体验：

- 支持中英文界面切换。
- 支持账号登录状态展示。
- 支持模型选择与模型目录。
- 支持桌面端更新检查与安装。
- 支持导出诊断包。
- 支持重启本地后端运行时。
- 支持项目、Workflow 和运行记录的本地管理。

## 内置起点

0.1.0 默认包含一组基础 Agent 与 Workflow，用于帮助用户快速开始：

- Manager：负责人类沟通与工作流管理。
- Architect：负责架构设计、任务拆解与评审。
- Developer：负责编码与具体执行。
- Tool Tester：用于工具能力验证。
- Simple Dev Workflow：一个基础开发协作 Graph。

这些不是最终答案，而是起点。The One 的重点不是让所有人使用同一套 Agent，而是让每个人构建自己的多智能体架构。

## 已知边界

0.1.0 是初始版本，重点是验证产品核心闭环：仓储、组装、运行、观察和动态协作。

当前版本更适合愿意探索 Multi-Agents 架构的开发者、研究者和高级用户。部分高级能力，尤其是运行时 Graph 扩展、Task Board 分层协作和 MCP 配置，仍然需要用户理解 Agent 架构本身。

## 总结

The One 0.1.0 交付的是一个清晰的起点：

你可以创建 Agent，组合工具和 Skill，搭建 Multi-Agents Graph，在本地项目中运行它，并实时观察 Agent 之间如何沟通、思考、调用工具和协作完成任务。

这就是 The One 的第一步：把 Multi-Agents 从代码实验，推进到可视化、可运行、可调试、可演化的桌面工作台。
