---
title: Blueprints
description: Visual workflow graphs that bind nodes to agent presets.
---

A blueprint is the workflow graph.

Nodes represent agent execution points. Edges represent routing, handoff, and execution order.

## What belongs in a blueprint

- Agent nodes.
- Edges between agents.
- Handoff points.
- Parallel branches.
- Runtime orchestration affordances.

## What does not belong

Do not hide important workflow logic in a long prompt. If routing matters, make it visible in the graph.
