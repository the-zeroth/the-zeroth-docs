---
title: Local Execution
description: Launch a blueprint against any workspace on your machine.
---

Local execution means the workflow runs with direct access to the workspace you choose.

## Workflow

1. Choose a blueprint.
2. Select a local path.
3. Start the run.
4. Observe agent actions and tool calls.

## Why it matters

Agents should work where the code, files, and project context actually live.

## Boundary

Local execution should be explicit. Do not grant workspace access casually.
