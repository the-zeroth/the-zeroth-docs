---
title: Runtime Orchestration
description: Let agents reshape workflow execution while a run is alive.
---

Runtime orchestration is what makes the workflow more than a static graph.

Agents can hand off state, update plans, change execution shape, and spawn parallel subtasks while the run is active.

## Common operations

- Structured handoff.
- Plan update.
- Parallel subtask creation.
- Routing changes.
- Human intervention.

## Design rule

Runtime changes must stay observable. If the workflow changes shape, the user should be able to see why.
