title: The One 0.2.0 Release Notes
description: 0.2.0 带来 Provider 接入、Definition Space、分享平台和 Graph 语义升级。
---

The One 0.2.0 是一次关键版本更新：你现在可以接入更多 LLM Provider，管理多套完全独立的 Definition Space，并通过分享平台交换自己的 Agent / Skill / Graph 组合。

同时，我们正式将用户界面中的 “Workflow” 语义调整为 “Graph”。这不是简单改名，而是对 The One 产品理念的纠偏：我们的 Multi-Agent 架构不是线性的流程，而是由多个 Agent 通过自由通讯关系组成的协作图。

## 核心变化

### 更开放的 LLM Provider 接入

0. 2.0 大幅扩展了 The One 的模型接入能力。

除了 The One 官方订阅的模型调用路径，现在 The One 开始支持更丰富的第三方 LLM API 与连接方式，包括：

- OpenAI
- OpenAI Codex
- Anthropic
- Claude Code
- Google Gemini
- Gemini CLI Code Assist
- Gemini Antigravity Code Assist
- OpenRouter
- 自定义 OpenAI-compatible Provider

这意味着用户可以根据不同 Agent 的角色，为它们配置更合适的模型来源。架构师 Agent、执行 Agent、工具测试 Agent、研究型 Agent，不再必须绑定同一种模型策略。

0. 2.0 也加入了更完整的 Provider 配置体验：

- 在设置中集中管理模型服务连接。
- 支持 API Key 与 OAuth 类型的连接方式。
- 支持按 Provider 查看和筛选可用模型。
- 支持模型目录刷新与连接感知的模型选择。
- 支持自定义 Base URL，用于兼容更多 OpenAI-compatible 服务。

这一步很重要。Multi-Agent 系统的上限不只取决于 Graph 怎么连，也取决于每个 Agent 背后使用什么模型、什么能力、什么调用路径。0.2.0 让 The One 的模型层开始真正变得可组合。

### 多 Definition Space

0. 2.0 引入了 Definition Space。

一个 Definition Space 是一整套独立的多智能体资源集合，包含：

- Agent
- Prompts / Profiles
- Skills
- MCP 配置
- Graph Blueprints
- Requests

你可以把它理解为“一个完整的 Multi-Agent 架构包”。不同 Space 之间互不污染，可以独立创建、复制、切换、导入和导出。

这解决了 0.1.0 里的一个明显边界：所有定义资源都在同一套仓储里，适合开始探索，但不适合长期维护多个完全不同的 Agent 系统。现在你可以为不同用途维护不同 Space，例如：

- 一个用于软件开发的 Agent Graph 集合。
- 一个用于研究和资料分析的 Agent Graph 集合。
- 一个用于自动化办公的 Agent Graph 集合。
- 一个用于实验新 Skill / MCP 组合的沙盒 Space。

切换 Space 时，The One 会切换当前使用的 definitions，让你在不同多智能体架构之间快速切换，而不是手动搬文件、改配置、怕串环境。手动管理这类东西是典型的低级错误温床，0.2.0 开始把它产品化了。

### 分享平台与 Space 交换

Definition Space 不只是本地隔离。0.2.0 还打通了官网分享平台。

你现在可以：

- 将本地 Space 导出为 `.zeroSpace` 包。
- 将 Space 上传到官网分享平台。
- 设置分享可见性。
- 为分享内容添加标签。
- 从分享平台下载 Space。
- 通过深链导入他人分享的 Space。
- 在导入前预览 Space 内容和潜在风险。

这让 The One 开始从“个人多智能体工作台”走向“Multi-Agent 架构社区”。以后用户分享的不应该只是一段 Prompt，而是一整套可以运行、可以观察、可以继续改造的 Agent Graph 系统。

我们也加入了必要的安全处理：导出 Space 时会清理明显敏感的 MCP 配置字段；导入外部 Space 时，会在真正落地前展示可疑脚本和结构校验信息。因为导入别人的 Skill，本质上就是把别人的可执行资源放进你的本机环境。这里不能装傻，必须让用户看清楚风险。

### Workflow 正式更名为 Graph

0. 2.0 将用户界面中的 Workflow 语义调整为 Graph。

原因很简单：Workflow 这个词容易误导用户，以为 The One 运行的是线性的、预设步骤式的流程。但 The One 的核心不是线性自动化，而是 Multi-Agent Graph。

在 The One 中：

- Agent 之间可以通过边建立通讯权限。
- Agent 可以唤醒其他节点。
- Agent 可以进入等待状态。
- Agent 可以形成子 Graph。
- Agent 可以在运行中扩展协作关系。
- 整个系统更接近动态协作网络，而不是单向流水线。

所以，“Graph” 比 “Workflow” 更准确。

0. 2.0 中，主要用户界面已经围绕 Graph 重新表述：

- Graph Preset
- Graph Blueprint
- Graph Orchestrator
- New Graph
- Graph inventory
- Runtime Graph
- subGraph

这次改名不是为了好看，而是为了避免错误心智模型。名字如果错了，用户理解产品的方式也会错。这个坑必须早填，拖到后面就是产品债务。

## 体验改进

### 设置页重构

0. 2.0 对设置页进行了重新组织，把桌面端设置拆得更清楚：

- 常规设置
- Provider / API 配置
- Definition Spaces 管理
- 桌面运行时状态
- 更新、诊断与后端控制

这让新增的 LLM Provider 和 Space 能力不至于塞成一坨。设置页如果继续乱堆，后面只会变成配置垃圾场，这次拆分是必要的。

### 模型选择体验增强

Agent Preset 中的模型选择现在更贴近 Provider 连接状态。模型列表不只是静态候选项，而是会结合当前配置的模型服务与连接信息展示。

这对于 Multi-Agent 架构很关键：不同 Agent 需要不同模型，不同模型又来自不同 Provider。0.2.0 开始让这个选择过程更清晰。

## 总结

The One 0.2.0 的重点是三个字：开放化。

它让 The One 可以连接更多模型服务，管理多套独立的 Multi-Agent 架构，并把这些架构打包分享到更大的社区中。

0. 1.0 证明了 The One 可以搭建、运行和观察 Multi-Agent Graph。  
0. 2.0 则开始证明：这些 Graph 可以被切换、复用、分发，并与更广泛的 LLM 生态连接起来。

The One 正在从一个 Multi-Agent 桌面应用，走向一个 Multi-Agent 架构平台。
